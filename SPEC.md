# Схематор

Схематор берёт пример JSON и помогает превратить его в чистую типовую схему на
языке **essensio**. Вы вставляете данные, даёте имена повторяющимся структурам —
и получаете готовое описание типов. Нотация следует грамматике
[essensio/notation](https://github.com/essensio/notation).

## Зачем он нужен

Когда у вас на руках только пример ответа какого-нибудь API, структура в нём есть,
но она безымянная. Глядя на массив заказов, человек понимает, что каждый его
элемент — это «Заказ», но в самом JSON такого имени нет. Схематор выводит структуру
автоматически и даёт удобно её назвать, чтобы из сырого примера получилась
переиспользуемая схема.

Возьмём такой ввод:

```json
{
  "orders": [
    { "address": "Машиностроителей 15", "date": "2024-04-23", "qty": 2 },
    { "address": "Ленина 1",            "date": "2024-04-24", "qty": 5 }
  ]
}
```

Сразу после вставки Схематор выводит типы, но все структуры пока безымянные:

```
{orders: {address: Строка, date: Строка, qty: Число}[]}
```

Элемент массива — `{address, date, qty}` — это **анонимный тип**, и Схематор его
подсвечивает. Вы кликаете по нему, вводите имя «Заказ» — и схема становится чистой:

```
Заказ = {address: Строка, date: Строка, qty: Число}

{orders: Заказ[]}
```

Теперь у структуры есть имя, на неё можно ссылаться, и описание читается как
настоящая схема, а не как распечатка примера.

## Анонимные типы

Анонимный тип — это структура без имени. В нотации essensio это безымянный
**кортеж** (JSON-объект) или **отношение** (массив объектов): на такой
тип нельзя сослаться, пока он не назван. Именно их Схематор подсвечивает и предлагает
назвать. Скаляры — число, строка, булево, `Пусто` — это системные типы; они всегда
именованы и в подсветке не участвуют.

Назвать анонимный тип — единственное действие разметки. Назвали — он поднимается
наверх отдельной декларацией `Имя = …`, а на прежнем месте остаётся ссылка по
имени. Не назвали — он так и остаётся встроенным в описание и подсвеченным,
напоминая, что его стоит назвать.

## Как выводятся типы

Каждое значение JSON получает тип по простым правилам:

| Значение JSON | Тип essensio |
|---|---|
| `null` | `Пусто` |
| `true` / `false` | `Булево` |
| число | `Число` |
| строка | `Строка` |
| объект | кортеж `{поле: тип, …}` |
| массив объектов | отношение `Кортеж[]` (поля объединяются) |
| массив однотипных скаляров | `Число[]`, `Строка[]` и т. п. |

Строка остаётся `Строка`: Схематор не угадывает из неё `Дата` или другие сужения —
это отдельное решение о типе, а не разметка имени.

**Объединение полей.** Если в массиве лежат объекты с **разным набором полей**,
Схематор сводит их к одному кортежу, объединяя все поля: поле, что есть лишь в части
объектов, включается и берёт тип оттуда, где присутствует. Так `[{a:1,b:2},{a:3}]`
даёт `{a: Число, b: Число}[]`.

Два пограничных случая массивов. **Пустой массив** не даёт узнать тип элемента —
Схематор помечает его как отношение с неизвестным элементом. **Сырьё** — это массив,
элементы которого слить нельзя (объект вперемешку со скаляром или скаляры разных
видов); типизировать и именовать там нечего.

## Похожие блоки и общее имя

Две структуры считаются одинаковыми, когда у них одна и та же форма — один и тот же
набор полей с одинаковыми типами. В примере выше оба заказа одной формы, поэтому
достаточно одного имени «Заказ» на обоих.

Схематор не объединяет одинаковые блоки сам. Когда вы называете один блок, он
показывает, сколько ещё блоков той же формы есть в документе, и предлагает кнопку
**«Применить ко всем похожим»**. Так одинаковые формы можно назвать одним именем
одним движением — но только если вы этого захотели. По умолчанию имя получает
только тот блок, который вы назвали.

Одно имя описывает одну форму. Если попытаться назвать одним именем две разные
формы, Схематор покажет конфликт. Обратное допустимо: две одинаковые формы можно
при желании назвать по-разному.

## Состояние и пересчёт

Всё, что вводит пользователь, — это текст JSON и присвоенные имена. Больше ничего не
хранится. Выведенные типы, список анонимных блоков, подсветка и текст схемы — всё
это пересчитывается из текста и имён при каждом изменении. Один источник истины,
поэтому рассинхрона между «данными» и «схемой» не бывает в принципе.

Пересчёт детерминирован: одни и те же ввод и имена всегда дают одну и ту же схему,
без обращения к часам, случайности или внешним данным.

## Краевые случаи

Схематор корректно обрабатывает: пустой ввод; невалидный JSON (показывает ошибку с
позицией, схему не строит); пустой объект `{}` — это кортеж из нуля полей, его тоже
можно назвать; пустой массив; разнородный массив; глубокую вложенность; строковые
ключи вроде `{"order-id": 1}`; поле со значением `null` (тип `Пусто`); числа с
экспонентой; и попытку дать одно имя двум разным формам.

## Интерфейс

Экран — две панели.

Слева — **разметка**: ваш JSON, аккуратно отформатированный, с подсвеченными
анонимными блоками. Клик по блоку открывает поле ввода имени и, если есть похожие,
кнопку «Применить ко всем похожим».

Справа — **схема**: готовое описание essensio, только для чтения, моноширинным
шрифтом. Ещё не названные анонимные типы подсвечены здесь так же, как слева.

Сверху — кнопки «Пример», «Очистить» и «Копировать». Пустой экран показывает
подсказку и кнопку загрузки примера; невалидный JSON — сообщение об ошибке вместо
схемы.

Цвет работает осмысленно. Нейтральная гамма — `slate`, основное действие — `indigo`,
подсветка анонимного типа — мягкий жёлтый маркер (`amber`), читающийся как «здесь
тип без имени». Все размеры, отступы и цвета берутся из стандартных шкал Tailwind.

## Архитектура

Ядро (`src/core`) — чистый TypeScript без Vue: разбор JSON, вывод типов, сравнение
форм и генерация текста essensio. Оно покрыто тестами и ничего не знает про
интерфейс. Модель типов в ядре — это представление схемы essensio.

Интерфейс (`src/components`) — Vue 3. Он хранит только текст и имена, а всё
остальное получает из ядра через вычисляемые свойства.

Сборка — Vite с относительным базовым путём, чтобы собранная статика работала из
любой подпапки на хостинге демо.
