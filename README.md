# Схематор

JSON → схема **essensio**. Вставляешь пример JSON, размечаешь анонимные типы
именами — на выходе получаешь описание схемы в нотации essensio.

Демо: **https://essensio.github.io/demos/schemator/**

- Спецификация — [`SPEC.md`](SPEC.md).
- Нотация — [essensio/notation](https://github.com/essensio/notation).
- Печать схемы и проверка имён — пакет [`@essensio/engine`](https://github.com/essensio/ts-engine)
  (свою модель типов essensio приложение не заводит).

## Возможности

- Вывод типов из JSON: кортежи, отношения, скаляры; объединение полей у массива
  объектов с разным набором полей.
- Разметка = именование: анонимные кортежи/отношения подсвечиваются, по клику
  получают имя и поднимаются в декларацию (`Заказ = {…}`, `Заказы = Заказ[]`).
  Скаляры можно назвать как подтипы-домены (`Количество = Число`).
- «Применить ко всем похожим» для одинаковых форм.
- Примеры с deep-link (`?example=<uuid>`).

## Разработка

```bash
npm install
npm run dev      # локальный сервер
npm test         # vitest
npm run build    # сборка статики (vite, base: './')
```

Стек: Vue 3 + TypeScript + Tailwind CSS.

> Зависимость `@essensio/engine` подключена как `file:../ts-engine` — нужен
